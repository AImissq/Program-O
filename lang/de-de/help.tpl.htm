Program-O Installationshilfe
[helpContent]
 
Was das alles bedeutet...
Diese Datei soll dir dabei helfen die verschiedenen Einstellungen die du während der Installation vornehmen kannst zu verstehen. Jeder Abschnitt wird erklärt und wenn nötig mit Beispielen erklärt. Einige Einstellungen werden automatisch generiert, dennoch solltest du genau hinsehen. Bitte besuche die Seite Program O.
 
Allgemeine Konfiguration
 
Botmaster Information
Dein Name
Selbsterklärend. Gib deinen Namen ein. Mit welchen Namen sollen dich deine Besucher ansprechen?
E-Mail-Adresse
Wird intern genutzt für Kontaktdetails und für das Übertragen von Debug-Code. Das Übertragen von Debug-Code ist optional. Für mehr Informationen siehe unten, in den Debug Informationen.
 
Bot Konfiguration
Ebenfallas selbsterklärend. Folgende Einträge sind vorhanden:
Bot Name
Ohne Namen geht es nicht. Wähle deinen Bot Namen weise.
Bot Beschreibung
Gebe deinen Bot eine Hintergrundgeschichte beschreibe Ihn wenn möglich. Dieses Feld muss aber nicht unbedingt ausgefüllt werden. Du kannst es auch leer lassen.
Bot Aktivieren
Du kannst deinen Bot als versteckt markieren, dies kann in einigen Bereichen von Nutzen sein. Aktiviere diese Option wenn du deinen Bot normal nutzen möchtest. Ändere dies am Besten auf der Administrator-Seite.
Fehlerbenachrichtigung
Diese Antwort wird ausgegeben, wenn ein interner Fehler auftritt. Für was du dich entscheidest liegt ganz bei dir, es ist zu empfehlen irgendein Wert auszusuchen.
Standard Pattern Standardwert: *
Dieser Wert sollte nicht geändert werden und ist mittlerweile veraltet.
Anzahl der Chat-Zeilen
Wie viele Zeilen sollen im Chat angezeigt werden? Setze es auf 0 für unendlich. Beachte dass dies die Performance deines Bots beeinflussen kann, wenn viele Benutzer gleichzeitig agieren.
Anzahl der Antworten im Verlauf
Bestimmt wie viele Antworten jeweils zwischengespeichert werden, damit sich der Bot "zurückerinnern" kann. Diese Einstellung ist limitiert, das dies sonst zu viel Leistung kosten würde. Der Wert sollte bei mindestens 30 liegen. Solltest du JSON nutzen, setze diese Einstellung auf eins und implementiere deinen eigenen Verlauf client-seitig.
Antwort Format
Standard: HTML
Program-O gibt dir drei Optionen zur Verfügung um die Antworten deines Bots zu parsen:
  • HTML Diese Option bewirkt dass, Program-O HTML als Ausgabeformat nutzt. Für Neulinge ist diese Option vorzuziehen, da diese am wenigsten Probleme macht.
  • JSON Diese Option bewirkt dass, Program-O JSON als Ausgabeformat nutzt. Diese Option ist vorzuziehen, wenn dein Bot client-seitig angesprochen werden soll, so ist es relativ einfach möglich mithilfe von jQuery Anfragen zu verschicken.
  • XML Diese Option bewirkt dass, Program-O XML als Ausgabeformat nutzt. Standardisiertes Format und leicht auszulesen so wie breite Unterstützung.
Hier gibt es einige Beispielseiten zum ausprobieren. Vergesse aber nicht, dass diese Seiten nur zum experimentieren sind. Für den produktiven Einsatz sind Modifikation unerlässlich.
 
Datenbank Konfiguration
 
Info zur Datenbankverbindung
Bitte beachte, dass du bereits im Vorfeld eine Datenbank für deinen Bot anlegen musst. Die nötigen Tabellen werden automatisch angelegt. Aus dem Grund dass, viele Hoster das anlegen von Datenbanken aus dem Skript nicht erlauben. Verbindungsinformation wie z. B. der Benutzername und Passwort sollten daher vorliegen.
Datenbank Host Name
Standard: localhost
Standardmäßig localhost, es kann aber auch sein, dass dein Host dir einen anderen Host Name zuweist. Diese Eingabe ist erforderlich.
Datenbank Port
Standard: 3306
Nur erforderlich wenn der Server so konfiguriert wurde, dass ein anderer Port genutzt wird.
Datenbank Name
Wenn du bereits eine Datenbank angelegt hast, gebe diesen Namen bitte hier ein.
Datenbank Benutzername
Der Name des Benutzers welche Rechte für die neu angelegte Datenbank besitzt.
Datenbank Passwort
Das Passwort welches zum Anmelden deiner Datenbank benötigt wird. Achte darauf dieses Feld auszufüllen.
Datenbank Zeitzone
Die Zeitzone deines Servers. Diese Einstellung wird automatisch vorgenommen, sollte aber noch einmal überprüft werden.
 
Administration
Administrator Benutzername
Zum Anmelden wird ein Benutzername und Passwort benötigt. Das ist der Benutzername des Administrator.
Administrator Passwort
Genau wie beim Benutzernamen. Diese Einstellung ist erforderlich. Das Passwort wird als Hash in einer Datenbank gespeichert.
 
Debug Optionen
Debug Stufe
Diese Einstellung bestimmt die Menge der Ausgaben welche von den Debug-Funktionen erzeugt wird. Du hast die Wahl zwischen vier Optionen:
  • Keine Es werden keine Debug-Daten angezeigt und kann die Performance des Bots steigern.
  • Allgemein Es werden nur allgemeine bis wenig Informationen generiert.
  • Allgemein/SQL Kleiner Zusatz bei dem zusätzlich SQL-Anfragen gespeichert werden.
  • Alles Es wird eine umfangreiche Berichterstattung generiert und jeder Funktions-Aufruf wird protokolliert. WARNUNG! Eine GROßE Menge an Daten wird dabei generiert, dass kann die Performance negativ beeinflussen, gerade dann wenn die Daten im Quellcode angezeigt werden. Vermeide diese Option, wenn du Debug-Informationen zu deiner E-Mail geschickt bekommen möchtest.
In den meisten Fällen sollte die Option auf "Keine" stehen, außer du möchtest ein Problem zurückverfolgen. Die letzten zwei Optionen dienen allein der Fehlerbehebung.
Debug Modus
Standard: Datei
Mit "Debug Modus" wird das Ausgabeformat der Debug-Informationen festgelegt.
  • Im Quellcode anzeigen Debug-Informationen werden im Quellcode angezeigt, innerhalb von HTML Kommentaren. Hinweis - Funktioniert nur wenn das Ausgabeformat auf HTML gestellt ist(Siehe weiter unten)
  • Datei Debug-Informationen werden in einer Datei chronologisch gespeichert. Der benutze Ordner liegt in /chatbot/debug/.
  • Anzeigen Zeigt Debug-Informationen direkt auf dem Bildschirm an. Hinweis - Funktioniert nur wenn das Ausgabeformat auf HTML gestellt ist(Siehe weiter unten)
  • E-Mail Sendet Debug-Informationen via E-Mail - WARNUNG! Das kann eine MENGE E-Mail Traffic verursachen! Nutze es Weise.
Sicherung
Standard: Session
Diese Einstellung bestimmt wie Program-O Benutzereingaben speichert. Hier kannst du zwischen folgenden Optionen wählen:
  • Session Das ist der Standard. Alle Daten werden in einer einzelnen Session-Variable zwischengespeichert. Je nach Einstellungen des Browsers, erlöscht die Session nach dem Schließen des Browsers.
  • Datenbank Anstatt die Daten in einem Array zwischenzuspeichern, werden hier die Daten in einer Datenbank gespeichert.
Hier gibt es keine Option die vom Vorteil wäre. Wähle die Option die dir am meisten zusagt.
 
Fehlerbehebung
Datei Zugriffsrechte

Solltest du diese Seite sehen, dann weil dein Server dem Installations-Skript keine Rechte gewährt. Sollte dies der Fall sein, dann ändere die Zugriffsrechte der einzelnen Ordner und Dateien. Damit die Konfiguration abgeschlossen werden kann.

Um manuell die Zugriffsrechte auf deinen Server zu ändern, gibt es mehrere Wege. Sollte dein Host eine Benutzerkontrolle haben, kannst du die einzelnen Zugriffsrechte anpassen oder du benutzt eine FTP-Software welche dir ebenfalls erlaubt Rechte zu ändern. Für weitere Informationen lese die Hilfs-Seite deines FTP-Programms und suche nach "CHMOD" oder "Zugriffsrechte".

Die korrekt Einstellung für die Zugriffsrechte ist 0755. Wenn du nicht weißt was damit gemeint ist, dann werfe einen Blick hierauf http://www.perlfect.com/articles/chmod.shtml. Merke dir einfach nur, dass diese Einstellung festlegt ob die Konfigurationsdatei beschrieben werden kann oder nicht und es weiterhin möglich ist von der Datei zu lesen. Wenn auf diese Datei nicht geschrieben werden kann, können die Einstellungen nicht gespeichert werden welche du vorgenommen hast. Dein Bot würde also nicht richtig funktionieren. Es ist also äußerst wichtig.
